03MCP tools · the LLM surface

Five tools. Every hit carries a canonical_url. Every tool defaults to markdown for rendering; pass format="json" for machine-parseable output.

search Hybrid BM25 + Vector
Top-k chunks across the corpus, RRF-fused. 5-year recency half-life applies by default.
search( query: str, k: int = 8, types: list[str] | None = None, # ["Public_rulings", "Cases"] or ["Public_*"] date_from: str | None = None, # "YYYY-MM-DD" date_to: str | None = None, doc_scope: str | None = None, # doc_id glob, e.g. "TXR/TR20243/*" mode: "hybrid" | "vector" | "keyword" = "hybrid", sort_by: "relevance" | "recency" = "relevance", format: "markdown" | "json" = "markdown", ) -> str
  • types accepts exact matches and globs in the same list — ["Public_*", "Cases"] composes OR.
  • doc_scope is always a glob over the full doc_id path.
  • sort_by='relevance' applies a 5-year-half-life recency boost (tax law ages). 'recency' sorts strictly by date.
  • Returns: doc_id, title, type, date, heading_path, anchor, snippet, canonical_url, chunk_id.
search_titles FTS5 · title-only
Direct citation / case-name lookup. No body, no embeddings.
search_titles( query: str, k: int = 20, format: "markdown" | "json" = "markdown", ) -> str
  • Preferred when the query IS an identifier ("TR 2024/3", "PepsiCo", "Acts Interpretation Act 1901 s 1").
  • Since citations are inlined in title, a simple FTS match resolves them.
get_document Outline · Section · Paginate
One tool for all document reads.
get_document( doc_id: str, format: "outline" | "markdown" | "json" = "outline", anchor: str | None = None, heading_path: str | None = None, from_ord: int | None = None, # positional cursor include_children: bool = False, count: int | None = None, max_chars: int | None = None, ) -> str
  • No selector + outline → TOC table with start_ord/chunk_count/bytes. Cheapest way to survey a doc.
  • heading_path → grab a section by its breadcrumb; include_children=True rolls up the full subtree.
  • from_ord + count/max_chars → walk forward by position. Truncated JSON carries a continuation_ord.
get_chunks By id
Expand specific chunks (from earlier search results) to full text.
get_chunks( chunk_ids: list[int], format: "markdown" | "json" = "markdown", ) -> str
whats_new Recency feed
Newest documents by date, optionally filtered by since or types.
whats_new( since: str | None = None, # "2026-01-01" limit: int = 50, types: list[str] | None = None, format: "markdown" | "json" = "markdown", ) -> str

04Template-based metadata engine

ATO pages fall into ~10 structural templates. We classify each page by its heading-shape signature, then run a positional extractor. The output is always just two fields — title and date — which lands 100% coverage on both.

OFFICIAL_PUB
Rulings / PCG / TA / PS LA / ATO ID.
TR 2024/3 — R&D tax incentive
CASE_H1 / CASE_H2
Modern judgements — party-v-party, neutral citation, or Re-X.
PepsiCo Inc v Commissioner of Taxation
HIST_CASE
Pre-modern cases. Year from docid *YYYY*.
Ex parte Walton, In re Levy
DIS
Decision Impact Statement. Prefix DIS: for disambiguation.
DIS: Commissioner of Taxation v Wood
ACT
Full Act / Regulation title in H1.
Income Tax Assessment Act 1997
LEGISLATION_SECTION
PAC / REG docs — one section of an Act or Regulation.
Acts Interpretation Act 1901 s 10A
BILL_EM
Bills, Explanatory Memoranda, Second Reading Speeches.
EM to Banking Bill 1974
SMSFRB
SMSF Regulator's Bulletin — citation anywhere in the heading tree.
SMSFRB 2020/1
EPA
Edited Private Advice — ATO auth number is the identity.
EV 1051375298526
OTHER → universal fallback
Anything else gets <prefix> <body> from the docid.
NEM 200615
Iterable without re-embedding. Metadata lives outside the vector column, so ato-mcp backfill re-runs the rule engine against the existing corpus in ~3 minutes on CPU (rowid-keyed FTS updates, no GPU). Push a better rule; watch coverage of "title" improve on the next backfill.

05LLM workflows · what a call looks like

"What does the ATO say about R&D tax incentives for software development?"
1
search(query="R&D tax incentive eligibility software development", types=["Public_rulings", "Practical_compliance_guidelines"], k=8)
Hybrid search. 5-year half-life boost is automatic under sort_by='relevance'. Returns chunks from TR 2024/3, PCG 2023/2, plus supporting ATO IDs — each with a full title like "TR 2024/3 — R&D tax incentive eligibility".
2
get_document(doc_id="TXR/TR20243/NAT/ATO/00001", format="outline")
Cheap TOC of the most relevant ruling.
3
get_document(doc_id="TXR/TR20243/NAT/ATO/00001", heading_path="Ruling › Eligibility", include_children=True)
Pulls the full Eligibility subtree. Model quotes with canonical_url.
"Summarise the PepsiCo case and its implications for royalty withholding."
1
search_titles(query="PepsiCo")
Title FTS resolves the case and its DIS directly — no body search needed.
2
get_document(doc_id="LIT/ICD_NSD1162of2022/00001", format="markdown", max_chars=6000)
Full DIS body, capped. The DIS is the ATO's own summary.
3
search(query="royalty withholding tax embedded license", date_from="2024-01-01", k=6)
Post-PepsiCo commentary only. Date filter keeps the model from quoting pre-decision guidance as authoritative.
"Find section 355-25 of the ITAA 1997 and show me subsection (1) only."
1
search_titles(query="Income Tax Assessment Act 1997 s 355-25")
The title inlines the citation ("... s 355-25") so FTS resolves it.
2
get_document(doc_id="PAC/19970038/355-25", format="outline")
See the subsection structure before paying for body text.
3
get_document(doc_id="PAC/19970038/355-25", heading_path="(1)")
Single-subsection fetch.
"What new ATO guidance has come out on crypto since January?"
1
whats_new(since="2026-01-01", limit=30)
Recency feed. Model skims titles and spots crypto-relevant rulings.
2
search(query="cryptocurrency capital gains digital assets", date_from="2026-01-01", sort_by="recency", k=10)
Content-scoped, recency-sorted.
3
get_chunks(chunk_ids=[431284, 431290, 431301])
Expand top snippets to full chunk context for quoting.
"Compare how TR 2013/3 and its 2024 successor treat commercial debt forgiveness."
1
search_titles(query="TR 2013 3")
Resolve the 2013 doc's doc_id via title FTS.
2
search(query="commercial debt forgiveness", doc_scope="TXR/TR20133/*", k=5)
Scope the search to a specific doc via its doc_id prefix.
3
search(query="commercial debt forgiveness", doc_scope="TXR/TR2024*", date_from="2024-01-01", k=5)
Glob the 2024 TR series. Parallel passages from both rulings, ready to diff.

06Design principles

Zero duplication
One identifier (doc_id), one label (title), one date. URLs synthesised, status inlined, type in the docid prefix.
Fewer, more powerful tools
5 tools; get_document covers outline + section + pagination through one signature.
Scope globs, not tool variants
types and doc_scope both accept globs. One tool, many slices.
Templates, not regex soup
Classify structural shape, then run a positional extractor. Robust to heading noise; easy to extend.
Metadata is cheap
Anything non-semantic lives outside the embedding column. Rules iterate in minutes on CPU, not hours on GPU.
Positional navigation
Every chunk has an ord. Outlines carry start_ord. Paginated reads return a continuation_ord.
